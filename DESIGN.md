## ตัวตนผลิตภัณฑ์บน UI (ส่วนเดียวที่ต้องอยู่ในทุกดีไซน์ — มาจาก PRODUCT.md)
ทุกตัวเลข/ข้อเท็จจริงสำคัญ ต้องแสดง (หรือเข้าถึงได้ทันที): **Source** (มาจากไหน+ลิงก์) · **Freshness** (ข้อมูล ณ วันที่) · **Confidence** (สูง/กลาง/ต่ำ) · **Sample** (ราคามือสอง = "จาก N ประกาศ") · **สถานะข้อมูล** (ไม่มี = บอกว่าไม่มี ไม่โชว์ 0 ปลอม — `DataStatus`) · แยกชนิดราคาให้เห็นชัด (ป้ายมือหนึ่ง ≠ โปร ≠ ประกาศมือสอง ≠ ซื้อขายจริง)
— จะแสดง**หน้าตายังไง**เป็นเรื่องของดีไซน์แต่ละทิศ · กฎเหล็กผลิตภัณฑ์ (ไม่จัดอันดับรวม ฯลฯ) → `PRODUCT.md`
มาตรฐานขั้นต่ำเสมอ: responsive · เข้าถึงได้ (contrast/aria)
